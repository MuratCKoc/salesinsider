Lessons Learnt

What went right, and wrong.

In every project there will be opportunities to better deelop. The below are just some of the key takeaways.


Know Your Libraries and have alternatives

It is often far too easy to just load up a library, without fully comprehending the capabilities and limitations of the library. In our case, we initially opted to use MLtend library which is a well known library for apriori analysis. The challenge with MLtend is that is not the most optimized and relies on brute force. So when we ran a code for 38,000 line items, it either took thirty minutes to run the code or would not run at all.

Know Your Datasets

If data science is a journey then your dataset are the boots to carry you there. A suboptimal dataset can put a stop to any project faster than a bad client, or a bad manager. In our case, the dataset had multiple items that were similar if not overlapping such as "brandy" and "liqour".

Iterate with Smaller Datasets

The most important aspect in coding is constant iteration, and the best way to do that is with smaller datasets. When we were initially compiling our code, we would use the full dataset would take 10-20 minutes to reach an output.

Know Your Migration Rules

Data migration is where you can encounter the most frustrating errors. During the Transformation phase we wanted to visualize our compiled data table into a set of graphs. However during our compilation we kept encountering invalid escape string errors.
